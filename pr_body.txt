💡 **What:**
Optimized the array filtering logic in `useStatisticsProcessing` by converting `selectedFuels` and `selectedBrands` lookup arrays into JavaScript `Set` objects before executing the filtering loop.

🎯 **Why:**
Previously, the code used `Array.includes()` inside `.filter()` operations over large datasets. `Array.includes()` has $O(N)$ time complexity, making the entire filtering loop $O(M \cdot N)$ where $M$ is the dataset size and $N$ is the number of selected filters. By explicitly constructing a `Set` prior to the loop, we reduce the lookup time to $O(1)$, resulting in an overall complexity of $O(M + N)$.

This provides significant CPU relief inside the heavily re-calculated `useMemo` block, especially when filtering across a large number of brands.

📊 **Measured Improvement:**
A standalone micro-benchmark was written and executed inside the Node runtime, simulating filtering a 50,000 item dataset 100 times.

- **Fuel filtering (3 selections):**
  - Array.includes: ~419ms
  - Set.has: ~228ms
  - **Improvement: ~45% faster**

- **Brand filtering (20 selections):**
  - Array.includes: ~864ms
  - Set.has: ~207ms
  - **Improvement: ~76% faster**
