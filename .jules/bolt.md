## 2025-05-05 - Avoid tuple array allocations in React hooks and data processing pipelines
**Learning:** Using `Object.entries().reduce()` or `Object.values().reduce()` in high-frequency React hooks (e.g., `useMemo`) or hot data processing loops (like `useStatisticsProcessing` and `calculate_filtered_defects`) causes significant memory allocation overhead due to intermediate tuple arrays. In Node.js 22 sandbox benchmarking, replacing these with native `for` loops iterating over `Object.keys()` proved to be ~3x to ~7x faster.
**Action:** Default to `Object.keys()` combined with a native `for` loop for iterations over statically keyed objects in hot paths. Avoid higher-level functional abstractions like `Object.entries().reduce()` when processing large datasets or within frequent re-renders unless tuple arrays are specifically required.
