## 2024-05-18 - Avoid tuple arrays and reduce overhead in hot React hooks
**Learning:** In hot loops where performance matters (like within `useMemo` aggregating large datasets), iterating over objects using `Object.values().reduce()` or `Object.entries()` is significantly slower than standard iteration patterns. Benchmarking showed `Object.keys()` combined with standard `for` loops avoids the overhead of tuple array allocations and higher-order callbacks, making it dramatically faster. Specifically, replacing `for...in` and `hasOwnProperty` checks or `Object.entries` with `Object.keys(obj)` iteration provides a fast and clean way to traverse dictionary-like objects.
**Action:** When mapping, aggregating, or cloning objects within performance-critical hooks like `useStatisticsProcessing`, use `const keys = Object.keys(obj); for (let i = 0; i < keys.length; i++)` to eliminate callback and allocation overhead instead of high-level array abstractions.
