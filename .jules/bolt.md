## 2024-06-25 - Avoid Object.entries() in high frequency React hooks and O(N*M) Mapping
**Learning:** In a high-frequency loop within React data processing pipelines (such as evaluating defect codes for large vehicle lists), using `Object.entries()` or `Object.values().reduce()` can cause substantial garbage collection overhead due to tuple array allocations, and iterating over breakdown metrics on a per-item basis scales terribly (O(N * M)). Benchmarking indicates that using a native `for` loop on `Object.keys()` is significantly faster.
**Action:** When calculating mapped values in loops handling large lists, pre-calculate ratios or other static dependencies directly into a `Map` prior to iterating. Avoid `Object.entries()` and `Object.values()` when iterating large/dynamic objects in these contexts, sticking to `.keys()` combined with traditional loop iteration.
