## 2023-10-24 - Consolidated Array Iterations in Data Processing
**Learning:** Chaining multiple `.map()` and `.filter()` operations over thousands of items in Next.js hooks (like `useStatisticsProcessing`) causes significant garbage collection overhead and redundant iterations. Furthermore, nested heavy helper calls (like `aggregateAgeRange`) were un-memoized during iteration, causing them to recalculate identically up to 3 times per item.
**Action:** When processing large arrays in JavaScript, consolidate multiple mapping and filtering passes into a single `for...of` loop. Calculate heavy helper values once per iteration, store them in a local variable, and reuse them to construct the final array without allocating intermediate objects.

## 2023-10-25 - Avoid Object.entries/values in React Hot Loops
**Learning:** Using `Object.entries()` or `Object.values()` inside performance-critical data processing loops (like `useStatisticsProcessing` aggregating stats over tens of thousands of vehicles) creates immense garbage collection overhead. These native Object methods allocate intermediate arrays per item in the loop, severely slowing down Next.js hydration and filtering on large datasets.
**Action:** When iterating over dictionaries or checking values in heavily executed hot loops or deeply nested hook callbacks (e.g., `calculate_filtered_defects`), replace `Object.entries/values` with manual `for...in` loops. This avoids temporary array allocation entirely and significantly improves execution time, cutting down loop processing by ~4x in micro-benchmarks.
