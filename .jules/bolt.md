## 2024-05-27 - Polars Native Dict Conversion
**Learning:** Polars `.iter_rows()` vs native dict building. In modern Polars versions (1.4+), iterating rows via `.iter_rows()` can sometimes be faster than natively zipping `df.get_column("col")` when struct elements are extracted into Python dicts, because `iter_rows` effectively translates rust structs to Python objects cleanly before loop entry. However, replacing explicit loops with zipped generator expressions (`dict(zip(..., ({...} for ...)))`) reduces Python-level array/tuple allocations and is explicitly preferred as a scaling optimization in memory-constrained environments where tuple materialization overhead hurts.
**Action:** Always prefer native dictionary comprehension and python column iterators (`df.get_column()`) over `.iter_rows()` for extracting mapping dictionaries from dataframes, but maintain awareness of actual overhead in the specific sandbox runtime if measuring absolute milliseconds vs memory safety.
